# Ralph + port offsets

This repo supports running multiple copies of the stack in parallel via `.env.worktree`.

## Port offset model

- Base ports stay stable in code/docs.
- Actual local ports are `base + offset`.
- All ports are defined in `.env.worktree` (gitignored, generated per worktree).

### Base ports

| Service | Base | Env var |
|---|---:|---|
| Postgres (host published) | 5432 | `PG_PORT` |
| Backend API (Hono) | 8080 | `API_PORT` |
| Frontend Web (Vite) | 4000 | `WEB_PORT` |
| Landing (Astro) | 4321 | `LANDING_PORT` |

## How ports work

All ports are stored in `.env.worktree` at the repo root. Generate it with:

```bash
# For main worktree (offset 0 + PROJECT_OFFSET)
./scripts/generate-worktree-env.sh --name main

# For feature branch (offset from branch name hash)
./scripts/generate-worktree-env.sh --name my-feature

# Manual offset
./scripts/generate-worktree-env.sh --offset 200
```

URLs are also generated:
- `API_URL=http://localhost:8080`
- `WEB_URL=http://localhost:4000`
- `LANDING_URL=http://localhost:4321`

## Worktree setup

When creating a new worktree, `.env.worktree` is automatically generated by `scripts/worktree/setup.sh`.

## Running commands

Use `scripts/with-worktree-env.sh` to load `.env.worktree` before running commands:

```bash
./scripts/with-worktree-env.sh pnpm dev
./scripts/with-worktree-env.sh just db-migrate
```

## Docker Compose

`compose.yml` reads `PG_PORT` from the environment:

- Postgres publishes `${PG_PORT}:5432`

From host: connect to `localhost:$PG_PORT`
From containers: use `postgres:5432` (fixed internal port)

## Ralph scripts

Ralph scripts accept an optional `PRD_NAME` (default: auto-detected from branch).

They generate `.env.worktree` with an offset derived from the PRD name.

- `scripts/ralph/setup.sh <PRD_NAME>` - Run docker compose + migrations + seeds
- `scripts/ralph/start.sh <PRD_NAME>` - Start dev servers in background
- `scripts/ralph/health-check.sh <PRD_NAME>` - Verify services are ready
- `scripts/ralph/teardown.sh <PRD_NAME>` - Stop servers and cleanup

Examples:

```bash
./scripts/ralph/setup.sh inventory-management
./scripts/ralph/start.sh inventory-management
./scripts/ralph/health-check.sh inventory-management
./scripts/ralph/teardown.sh inventory-management
```
